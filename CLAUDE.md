# PokeMatchup — Claude Context

## What this is

Local single-user battle advisor for Pokémon Unbound (and future game profiles). During gameplay: pick the opponent → get ranked team recommendations with visible damage math. No auth, no server, no cloud. Runs on `localhost` next to the emulator.

## Tech stack

- React 18 + Vite + TypeScript (strict mode)
- Tailwind CSS (dark theme, gray-900/gray-950 bg)
- Zod — schema validation for localStorage reads and OCR API responses
- Vitest — unit tests for `damage.ts` and `dex.ts` only
- Anthropic API (`claude-sonnet-4-5`) — screenshot OCR, key in `.env` as `VITE_ANTHROPIC_API_KEY`
- localStorage — all persistence (teams, corrections, active game)
- `tsx` devDep — runs `scripts/*.ts` directly via `npx tsx`

## Critical conventions

### Data access — always through dex.ts
```
src/lib/dex.ts  ← ONLY file that imports from src/data/games/
```
Components and hooks get a `Dex` instance from `useGameProfile()`. Never import JSON files directly.

### All scoring in damage.ts
```
src/lib/damage.ts  ← ALL formulas, pure functions, no React, no localStorage
```
Tweak weights by editing `DEFAULT_SCORING_CONFIG` (W_OFF, W_DEF, SPEED_BONUS, ONESHOT_PENALTY).

### Two-tier correction model
- **Session overrides** — React state in `useOpponent`, cleared when opponent changes
- **Dex corrections** — localStorage, keyed `dexCorrections:<gameId>`, survive refresh
- Applied in order: `base dex → DexCorrections → SessionOverride → ResolvedOpponent`
- Never merge them into one structure. `corrections.ts` reads both separately.

### OCR is a suggestion
Every OCR field is `ConfidenceField<T> = { value: T, confidence: 'high'|'medium'|'low' }`.
Low confidence → yellow highlight. Editing any field promotes it to `'high'`.

### Type chart direction
`typeChart[attackType][defenderType]` = multiplier. Attacker is the outer key.
Ability immunities (Levitate, Flash Fire, Water Absorb, etc.) override this inside
`dex.getTypeEffectiveness()` — NOT in the damage formula itself.

### TeamMember.stats = actual in-game stats
Dex `baseStats` are used as defaults when adding a new species. The formula uses
`member.stats` directly (user's real in-game numbers, baked-in EVs/IVs/nature).

## Data setup for Unbound (one-time)

1. Download the raw scraper output (gitignored):
   `https://github.com/nMckenryan/Borrius-Pokedex-Scraper/raw/main/scraperData/borrius_pokedex_data.json`
2. Place at `scripts/borrius_pokedex_data.json`
3. Run `npx tsx scripts/transform-borrius.ts`
4. Output: 506 Pokémon (503 Borrius + 3 supplements), 585 moves

To add a Pokémon not in the Borrius dex: edit `scripts/supplement-pokemon.json` and re-run.
To add a move not in the Borrius dex: edit `scripts/supplement-moves.json` and re-run.

## Adding a new game profile (purely additive)

1. Create `src/data/games/<game-id>/` with `meta.json`, `pokemon.json`, `moves.json`, `type-chart.json`
2. Add one import + one push to `GAME_REGISTRY` in `src/data/games/index.ts`
3. Zero other code changes required

## Gotchas learned the hard way

- `borrius_pokedex_data.json` wraps everything: `[{ info: {...}, pokemon: [...] }]`. Not a flat array.
- `level_learned_at` in the raw JSON is a **string** (`"1"`, `"3"`), not a number.
- `createDex` is async — `useGameProfile` exposes `{ dex: Dex|null, isLoading }`. Always null-check before use.
- Dynamic import path must be a literal string prefix for Vite to bundle correctly:
  `import('../data/games/${gameId}/pokemon.json')` — this works because gameId is a known constant at runtime, not a truly dynamic chunk.
- The Borrius regional dex (503) ≠ all Pokémon in Unbound. Use supplement files for the rest.
